Test: Nested Slot Forwarding

Options are forwarded through multiple shadow boundaries via nested slots. The innermost <select> must still discover them.

1. Two-level nested slot forwarding

Options live in the light DOM of <my-section>, are slotted into its shadow DOM, then forwarded through <my-custom-select>'s slot into the inner <select>.

Shadow 1 Shadow 2 Light DOM <slot> 
<my-section>
  ── shadow root 1 ───────────────
  <my-custom-select>
    ── shadow root 2 ─────────────
    <select>
      <slot>  ← inner slot
      </slot>
    </select>
    ──────────────────────────────
    <slot>  ← outer slot (forwarding)
    </slot>
  </my-custom-select>
  ────────────────────────────────
  Light DOM children of my-section:
  <option value="a">Apple</option>
  <option value="b">Banana</option>
  <option value="c">Cherry</option>
</my-section>

SLOT CHAIN:
options → outer slot → inner slot → inside <select>

FLAT TREE (what select should see):
<select>
  <option value="a">Apple</option>
  <option value="b">Banana</option>
  <option value="c">Cherry</option>
</select>

2. Three-level nested slot forwarding

Three layers of shadow DOM: <my-app> → <my-section> → <my-custom-select>.

Shadow 1 Shadow 2 Shadow 3 Light DOM 
<my-app>
  ── shadow root 1 ───────────────
  <my-section>
    ── shadow root 2 ─────────────
    <my-custom-select>
      ── shadow root 3 ───────────
      <select>
        <slot>  ← innermost
        </slot>
      </select>
      ────────────────────────────
      <slot>  ← middle
      </slot>
    </my-custom-select>
    ──────────────────────────────
    <slot>  ← outermost
    </slot>
  </my-section>
  ────────────────────────────────
  Light DOM children of my-app:
  <option value="1">First</option>
  <option value="2" selected>Second</option>
  <option value="3">Third</option>
</my-app>

SLOT CHAIN:
options → outermost slot → middle slot → inner slot → <select>

EXPECTED:
select.options.length === 3
select.selectedIndex === 1  (Second)

3. Dynamic insertion into nested structure

Adding an option to the outermost light DOM should propagate through all slot layers and appear in the inner select.

Shadow 1 Shadow 2 Light DOM 
BEFORE:
<my-section>
  <my-custom-select>
    <select>
      <slot></slot>
    </select>
    <slot></slot>
  </my-custom-select>
  <option value="x">X-ray</option>
  <option value="y">Yankee</option>
</my-section>

AFTER click (appends to my-section light DOM):
<my-section>
  ...
  <option value="x">X-ray</option>
  <option value="y">Yankee</option>
  <option value="z">Zulu</option>  ← new
</my-section>

EXPECTED:
New option propagates through both slots
select.options.length: 2 → 3

4. Imperative selectedIndex through nested slots

Setting select.selectedIndex programmatically should select the correct slotted option, even across two shadow boundaries.

Shadow 1 Shadow 2 Light DOM 
<my-section>
  <my-custom-select>
    <select>             ← JS: sel.selectedIndex = 2
      <slot></slot>
    </select>
    <slot></slot>
  </my-custom-select>
  <option value="r">Red</option>    index 0
  <option value="g">Green</option>  index 1
  <option value="b">Blue</option>   index 2 ← selected
</my-section>

EXPECTED:
select.selectedIndex === 2
select.value === "b"
selectedOptions[0].textContent === "Blue"
