Test: Slotted Button, Optgroups & Imperative APIs

1. Slotted button replaces default trigger

In base-appearance select, the first child <button> becomes the trigger. When both a button and options are slotted, the select must distinguish them: the button is the trigger, and the options populate the dropdown.

Shadow DOM Button (trigger) Options (slotted) 
<my-custom-select>
  ── shadow root ──────────────────
  <select>
    <slot>
      ▼ assigned nodes (light DOM):
      <button>Pick a fruit</button>  ← trigger, NOT an option
      <option value="apple">Apple</option>
      <option value="banana">Banana</option>
      <option value="cherry">Cherry</option>
    </slot>
  </select>
  ─────────────────────────────────
</my-custom-select>

EXPECTED:
select.options.length === 3 (button excluded)
Button renders as select's trigger

2. Slotted optgroup with nested options

<optgroup> elements slotted from the light DOM should preserve grouping. Options inside a disabled optgroup should be disabled.

Shadow DOM Light DOM Disabled 
<my-custom-select>
  ── shadow root ──────────────────
  <select>
    <slot>
      <optgroup label="Fruits">
        <option value="apple">Apple</option>     index 0
        <option value="banana">Banana</option>   index 1
      </optgroup>
      <optgroup label="Vegetables" disabled>
        <option value="carrot">Carrot</option>   index 2, disabled!
        <option value="pea">Pea</option>         index 3, disabled!
      </optgroup>
      <option value="water">Water</option>     index 4, standalone
    </slot>
  </select>
  ─────────────────────────────────
</my-custom-select>

EXPECTED:
select.options.length === 5
options[2].disabled === true (via optgroup)
options[3].disabled === true (via optgroup)

3. Slotted button + optgroups combined

Button as trigger, options organized in optgroups — all slotted.

Button Options <slot> 
<my-custom-select>
  <select>
    <slot>
      <button>Choose a color</button>  ← trigger
      <optgroup label="Warm">
        <option value="red">Red</option>         index 0
        <option value="orange">Orange</option>  index 1
      </optgroup>
      <optgroup label="Cool">
        <option value="blue" selected>Blue</option>  index 2 ←
        <option value="green">Green</option>    index 3
      </optgroup>
    </slot>
  </select>
</my-custom-select>

EXPECTED:
select.options.length === 4 (no button)
select.value === "blue"
select.selectedIndex === 2

4. Imperative API: set value and selectedIndex

Setting select.value and select.selectedIndex imperatively should work correctly with slotted options.

Shadow DOM Light DOM 
<my-custom-select>
  <select>
    <slot>
      <option value="dog">Dog</option>    index 0
      <option value="cat">Cat</option>    index 1
      <option value="fish">Fish</option>  index 2
      <option value="bird">Bird</option>  index 3
    </slot>
  </select>
</my-custom-select>

JS OPERATIONS & EXPECTED RESULTS:
Step 1: sel.value = "fish"
  → value === "fish", selectedIndex === 2

Step 2: sel.selectedIndex = 3
  → value === "bird", selectedIndex === 3

5. Option's nearest ancestor select

A slotted option's .form, .index, and .selected properties should correctly reference the shadow-DOM select as if it were the option's ancestor.

Shadow DOM Slotted options 
<my-custom-select>
  <select>                 ← ancestor in flat tree
    <slot>
      <option id="opt5a">One</option>    .index=0  .selected=false
      <option id="opt5b" selected>Two</option>  .index=1  .selected=true
      <option id="opt5c">Three</option>  .index=2  .selected=false
    </slot>
  </select>
</my-custom-select>

KEY ISSUE:
DOM ancestor chain: option → my-custom-select → ...
(no <select> found!)

Flat-tree ancestor chain: option → slot → <select>
(select IS found ✓)

EXPECTED:
opt5b.selected === true
opt5a.index === 0
opt5b.index === 1

6. Dynamic optgroup insertion and removal

Adding/removing an <optgroup> (with options inside) to the light DOM should update the select.

Shadow DOM Light DOM 
INITIAL:
<my-custom-select>
  <select>
    <slot>
      <option value="solo">Solo option</option>
    </slot>
  </select>
</my-custom-select>

AFTER adding optgroup (JS appends to light DOM):
<my-custom-select>
  <select>
    <slot>
      <option value="solo">Solo option</option>
      <optgroup label="Animals">          ← new
        <option value="cat">Cat</option>
        <option value="dog">Dog</option>
      </optgroup>
    </slot>
  </select>
</my-custom-select>

EXPECTED:
options.length: 1 → 3 (after add)
options.length: 3 → 1 (after remove)
